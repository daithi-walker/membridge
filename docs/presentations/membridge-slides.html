The Problem

3 / 12

Many sessions, no memory

When you run Claude Code across dozens of iTerm tabs over days or weeks, you quickly lose track of which session was doing what.

claude --resume <id>

Resuming a session is powerful - but only if you know which UUID to resume. Without a tracker, you're guessing or grepping logs.

What MemBridge Does

4 / 12
  • 🔌Auto-registers every session via Claude Code hooks - zero manual setup per session
  • ✍️Auto-summarises what each session worked on when you stop, using Claude Haiku
  • 📊Dashboard - active / idle / stale status, git branch, prompt count, last active time
  • 🎯Focus button - jumps to the right iTerm2 tab, or opens a new tab with claude --resume
  • 🔗Session links - bidirectionally link related sessions; navigate across related work
  • 🔔Push notifications - macOS alert when Claude stops and needs your attention

How It Works - Hooks

5 / 12
UserPromptSubmit

Fires on every prompt - registers the session, captures project, branch, PID, iTerm UUID.

POST /api/heartbeat
PreToolUse

Fires before every tool call - keeps last_seen fresh during long responses.

POST /api/touch
Stop

Fires when the session ends - triggers async auto-summary via Claude Haiku.

POST /api/stop

All hooks run in the background (curl ... &) - Claude Code is never blocked.

Auto-Summary

8 / 12

When you stop a session, MemBridge reads the Claude Code transcript and asks Claude Haiku to write a short summary of what was accomplished.

Summaries are stored in an append-only log - you can see the full history of a session's work over time.

Trigger manually any time with /membridge-summarize.

# append-only, never overwrites

session_id  →  a3f9c1d2...
source      →  "auto"
summary     →  [added SSE push endpoint
             to FastAPI server; wired
             EventSource in app.js;
             removed 30s poll lag]
created_at  →  2026-06-26T09:41:00Z

Focus & Notifications

9 / 12

🎯 Focus button

One click raises the right iTerm2 tab. If the session is dead, opens a new tab and runs claude --resume <id> in the correct directory.

🔔 Awaiting input

When Claude stops and needs you - permission prompt, decision - the dashboard badge pulses and macOS fires a notification.

🔗 Session links

Link related sessions bidirectionally. Navigate between them from the side panel. /membridge-link creates links from the CLI.

Focus states: ◉ awaiting ◉ working ↩ resume ⌘ idle

Slash Commands

10 / 12
/membridge-summarize

Generate and push a summary for the current session mid-flight

/membridge-rename

Set a short description - Claude can self-tag its own sessions

/membridge-note <text>

Inject a freeform note verbatim into session history

/membridge-link

Link current session to another by ID prefix

/membridge-context

Load summaries, notes, and linked sessions back into Claude
